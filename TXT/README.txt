================================================================================
  TXT/  -  централизованное хранилище СТАТИКИ для CLI-CODES
================================================================================

ВСЕ файлы в этой папке -- обычный UTF-8 (без BOM). Редактируются хоть в Notepad,
хоть в VS Code, без участия нейросети.

----------------------------------------------------------------------------
Зачем
----------------------------------------------------------------------------
Чтобы менять названия, логотипы, списки моделей, ссылки на получение API-ключей
и прочую статику в ОДНОМ месте -- без редактирования .ps1 / .sh скриптов и
без нейросетевого рефакторинга.

Динамика (живые ответы API, state.json, хэши паролей, логи) живёт ТОЛЬКО в коде.
TXT/ содержит ТОЛЬКО то, что человек может менять руками.

----------------------------------------------------------------------------
Форматы файлов (ВАЖНО)
----------------------------------------------------------------------------

Универсальные правила для ВСЕХ .txt в этой папке:

  • Строки начинающиеся с # -- комментарии, игнорируются.
  • Пустые строки -- игнорируются.
  • Кодировка -- UTF-8 (без BOM). Можно редактировать в Notepad++.
    PS5.1 на Windows-1251 консоли может прочитать кириллицу крякозябрами.
    Notepad++ → меню «Кодировки → UTF-8 (без BOM)».
  • Перевод строки -- LF \n (не CRLF). Notepad++ → меню «Правка → EOL →
    Unix (LF)».
  • Если строка пуста в одном поле -- оставить пустым (не удалять разделитель).
  • Символ-разделитель `|` НЕ должен встречаться ВНУТРИ полей.
    Все поля разделены `|`. Если в label нужна вертикальная черта -- 
    перефразируйте label (например, "слэш" вместо "a | b").

catalogs/*.txt -- списки моделей
─────────────────────────────────
Формат:  internal-id | api-model-id | label | context-tokens | max-output-tokens

  • internal-id       -- каноническое имя БЕЗ префикса провайдера.
                         Пример:  glm51   (а НЕ "zai-glm51").
  • api-model-id      -- что реально уходит в HTTP-запрос к API.
                         Пример:  glm-5.1
  • label             -- что показывается человеку в TUI-меню.
  • context-tokens    -- размер контекстного окна (число). 0 если N/A.
  • max-output-tokens -- макс. токенов на ответ (число). 0 если N/A.

  Пример:
    gpt-5-nano|gpt-5-nano|GPT-5 Nano (OpenAI, agentic)|128000|16384

logos/*.txt -- ASCII-арт баннеры
────────────────────────────────
RAW: каждая строка файла -- это строка баннера.
  • Первые строки -- верх.
  • Последние строки -- низ.
  • Никаких заголовков / разделителей. Чистый ASCII.
  • Auto-detect: пустые строки внутри баннера игнорируются.

menus/*.txt -- пункты меню и подзаголовки
─────────────────────────────────────────
Формат:  key | value

  Пример (menus/installer.txt):
    1|Установка сразу ВСЕХ агентов  ← рекомендуется
    9|Полное удаление проекта с ПК (uninstall)
    X|Выход из мастера установки

Ключи (1, 9, X) НЕ менять! Они привязаны к коду.

----------------------------------------------------------------------------
Как работает fallback
----------------------------------------------------------------------------

ВСЕ скрипты читают TXT/ в первую очередь. Если:
  • файл TXT/.../foo.txt НЕ существует, или
  • в файле нет ни одной не-комментарной строки,

то launcher-tui.ps1 / install-full.ps1 / launcher-catalog.ps1 падают обратно
на ИНЛАЙН данные (старые массивы прямо в .ps1). Это значит:
  • удаление TXT/ НЕ ломает проект;
  • любой TXT/.../*.txt можно дополнять / править точечно;
  • обновление через `git pull` подтянет свежие TXT/ вместе со скриптами.

----------------------------------------------------------------------------
Что НЕ трогать (динамика)
----------------------------------------------------------------------------

TXT/ НЕ содержит:
  • HTTP-ответы API провайдеров
  • Состояние сессий (state.json в scripts/)
  • Реальные API-ключи (они в User environment, не в TXT)
  • Имена процессов / пути к бинарям npm / Node.js

----------------------------------------------------------------------------
Пример типичной правки
----------------------------------------------------------------------------

  Задача:    в меню установщика убрать «OpenClaude» под номером [6].
  Решение:   НЕ редактировать install-full.ps1!
             Вместо этого: TXT/menus/installer.txt строки [6] не трогать,
             а в install-full.ps1 убрать `$installOpenClaude = $true` ветку.
             (или, если хочется скрыть UI, временно)
             Изменить `6|Только OpenClaude` на `6|— ` (пусто).

  Задача:    добавить новую модель в B.AI.
  Решение:   открыть TXT/catalogs/bai.txt и добавить строку
             новый-id|новый-api-id|Моя модель|200000|8192
             Сохранить. Без перезапуска скриптов -- при следующем запуске
             TUI меню само подхватит.

  Задача:    обновить ASCII-баннер Qwen.
  Решение:   открыть TXT/logos/qwen.txt, стереть содержимое, вставить
             свой ASCII-арт (например, из https://patorjk.com/software/taag/).
             Сохранить в UTF-8 LF. Запустить qwen.cmd -- баннер обновится.
