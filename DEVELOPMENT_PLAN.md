# Текущее состояние проекта CLI-CODES  
Судя по описанию, проект **CLI-CODES** содержит скрипты для установки популярных терминальных AI-агентов (install.ps1, bootstrap.sh), сборку красивого TUI-меню с выбором провайдера и шаблоны запуска агентов через бесплатные/дешевые API. Для начала полезно провести инвентаризацию текущих компонентов: какие агенты уже поддерживаются, как устроены скрипты и TUI. Например, используйте `grep`/`find` по репозиторию, чтобы увидеть вызовы конкретных CLI (например, `codex`, `gemini`, `hermes` и т.п.). Важно проверить:  
- **Скрипты установки** – `install.ps1` (Windows) и `bootstrap.sh` (Linux) – правильно ли они ставят нужные пакеты, нет ли ошибок.  
- **Список агентов** – какие команды уже есть в меню; соответствуют ли они актуальным CLI-агентам. Например, возможно сейчас есть OpenAI Codex, Gemini CLI и др.  
- **Провайдеры** – какие API уже используются и поддерживаются (OpenAI, Yandex, HuggingFace и т.д.).  

Параллельно можно проанализировать слабые места: нет ли фиксированных путей или устаревших версий инструментов, есть ли автотесты или CI. Оптимизация установки (кеширование, проверка зависимостей) и расширяемость TUI (например, драйвер для новых пунктов меню) – важные точки улучшения.

## Агентные CLI-инструменты и модели  
CLI-кодинговые агенты – это AI-инструменты, которые работают в терминале, могут читать/редактировать файлы, выполнять команды и совершать действия. Список таких проектов приведён в [репозитории «Awesome CLI Coding Agents»]. Среди самых популярных (звёздность GitHub в скобках) выделяются:  
- **Claw Code** (★193k) – открытая реализация архитектуры Claude Code (открыт исходный код после утечки).  
- **Hermes Agent** (★187k) – CLI-агент от Nous Research с персистентной памятью, автоскейлингом навыков и поддержкой 300+ моделей.  
- **OpenCode** (★172k) – терминальный агент с поддержкой 75+ провайдеров, LSP-интеграцией и «privacy-first» дизайном.  
- **Gemini CLI** (★105k) – Google-агент на базе Gemini с инструментами для работы с репозиториями.  
- **Codex CLI** (★89.6k) – локальный агент OpenAI (ChatGPT/Codex) с TUI и возможностью запуска кода.  
- **Open Interpreter** (★63.9k) – инструмент для выполнения кода на Python/JS и других языках через естественный язык.  
- **Cline CLI** (★62.9k) – универсальный автономный агент для планирования, правки файлов и команд.  
- **Pi (pi-mono toolkit)** (★60.8k) – лёгкий «harness» для агентов с единым API, TUI, поддержкой навыков и протокола MCP.  
- **Goose** (★47.8k) – локальный расширяемый агент с возможностью тестирования и встроенной поддержкой MCP.  
- **Aider** (★45.9k) – агент в стиле парапрограммирования через `diff`/патчи, ориентирован на git-воркфлоу.  
- **Qwen Code** (★25k) – CLI-агент Alibaba, оптимизированный под модели серии Qwen.  
- **Roo Code CLI** (★24.2k) – многофункциональный агент с «архитекторским» режимом и возможностью чекпойнтов.  

Каждый из этих агентов предназначен для «agentic» работы: поддерживает планирование (chain-of-thought), вызов внешних функций (tool calling), выполнение команд в ОС и т.д. Например, Codex CLI умеет запускать git и кодовые скрипты, Hermes имеет плагинные навыки и доступ к web-инструментам, а Open Interpreter может запустить Python-код на лету. Из них **Hermes, Claw Code, OpenCode, Qwen Code** и **Open Interpreter** актуальны и активно поддерживаются, поэтому их добавление в CLI-CODES расширит функционал проекта.  

**Модели с функцией вызова инструментов (Function/Tool Calling).** Некоторые LLM обладают встроенной поддержкой «tool calling». К примеру, GPT-4, GPT-3.5 Turbo поддерживают функцию [вызова функций (function calling)] через OpenAI API. Anthropic Claude в последних версиях (Claude 3 Sonnet, etc.) также поддерживает интеграцию со сторонними инструментами (через свой API). Google Gemini (с платформы Google AI Studio) реализует открытый [MCP-протокол](https://developers.redhat.com/blog/2024/01/17/how-to-build-a-simple-agentic-ai-server-with-mcp) (Model Context Protocol), позволяющий подключать внешние сервисы (например, web-браузер) к модели. Среди открытых моделей: Llama 2/3, Mistral, Qwen, Grok и др. пока не имеют нативного JSON-формата для вызова функций, но могут имитировать это через прунинг ответов или внешние обёртки.  

*Функция (tool-calling) – структурированный способ расширить возможности LLM. Модель получает JSON-описание функции (имя, параметры), а затем генерирует строгое JSON-обращение к ней. На схеме показан процесс: LLM формирует запрос-функцию и получает ответ. Такой механизм позволяет агенту запрашивать актуальные данные (например, текущую погоду) или выполнять действия через API.*  

Например, с помощью вызова функций LLM может интегрироваться с **базами данных, API, браузером, инструментами генерации изображений и т.д.** Это подтверждается исследованиями: лидерборд BFCL (Berkeley Function-Calling Leaderboard) специально измеряет способность моделей правильно вызывать функции. На практике можно использовать JSON-API или грамматики (Outlines, Jsonformer и др.) для контроля формата ответов. 

## Доступные бесплатные и недорогие API-провайдеры  
Для каждого LLM-агента важно выбрать провайдер, у которого есть бесплатная квота или низкая стоимость. Ниже перечислены ключевые варианты (с примерами моделей):  

- **OpenRouter** – единый API-гейтвей, предоставляющий бесплатный доступ (до 20 запросов/мин, 1000 запросов/день при пополнении $10) к множеству моделей. Среди них есть `Hermes 3 (Llama 3.1)`, `Llama 3.2 3B`, `Llama 3.3 70B`, `Dolphin-Mistral 24B`, `DeepSeek v4`, `Google Gemma 4 26B`, `Nemotron 3 120B`, `gpt-oss-120B` и др. OpenRouter особенно полезен для кластеров многообразных моделей без собственного API-ключа.  

- **Google AI Studio** – официальный сервис с бесплатным доступом к Gemini (3.5 Flash, 3.1 Flash-Lite, 2.5 Flash и их Lite-версии). Например, Gemini 3.5 Flash даётся 20 запросов в сутки (250k токенов/мин), Gemini 3.1 Lite – 500 запросов/день. Поддерживается также Google Gemini Robotics, модели Gemma от Google (3, 12, 27B) и др. Учётная запись Google Cloud/AI Studio даёт бесплатный начальный кредит.  

- **NVIDIA NIM** – облако NVIDIA с бесплатной регистрацией (требуется номер телефона). Доступны все популярные модели от NVIDIA (Llama, Nemotron, др.), но с ограничением 40 запросов в минуту и сравнительно малым контекстом.  

- **Mistral.ai (La Plateforme)** – сервис от Mistral с бесплатным экспериментальным планом (оформив телефон). Позволяет запускать модели Mistral (большие и малые) через API с лимитами ~500k токенов/мин. Ежемесячный лимит – миллиард токенов. Существуют также «Codestral» подписки и локальный «Mistral AI Gateway» для удобной работы.  

- **Hugging Face Serverless Inference** – бесплатные $0.10 кредитов/месяц для небольших моделей. Поддерживает запуски open-source моделей (Llama, Mistral, Vicuna и др.) в «Serverless» режиме (подойдут модели <10GB без сильно нагружающего inference).  

- **Vercel AI Gateway** – прокси/API-шлюз, через который можно достучаться до OpenAI, Claude, Qwen и др. моделей. Бесплатно даётся $5 в месяц. Позволяет маршрутизировать запросы разным провайдерам через единый ключ.  

- **OpenCode Zen** – API-шлюз проекта OpenCode (Anomaly), содержит набор собственных моделей (Big Pickle Stealth, Nemotron, DeepSeek) с бесплатным доступом.  

- **Cerebras Cloud** – NVIDIA-облако с бесплатной квотой (~60k токенов/мин у моделей gpt-oss-120B, Llama3-8B). Платформа Cerebras (Triton) даёт разовый миллион запросов/день при лимитах по запросам.  

- **Groq AI** – сервис Groq даёт бесплатный доступ к моделям Allam-2 7B и Llama3 8B (до 6000 токенов/мин).  

- **Alibaba Cloud (ModelStudio)** – бесплатный кредит 1M токенов для моделей Qwen (Llama-совместимых) и других. Позволяет работать с Qwen3, QwenCoder и др. через OpenAI-совместимый API (требует регистрации).  

- **Yandex Cloud AI Studio** – отечественный облачный провайдер. В Yandex есть *YandexGPT Lite, Pro и Alice AI* – GPT-подобные модели, специализированные на русском языке. У них бесплатные тестовые кредиты, поддерживаются OpenAI-совместимые интерфейсы. Для российских реалий это выгодно: юзер может получить начальный бесплатный токен и вызвать модели через привычный API-интерфейс Яндекса.  

- **Anthropic Claude** – хотя бесплатного API нет, Anthropic предоставляет бесплатные «research previews» (ограниченный бесплатный доступ) к своим моделям (Claude 3 Sonnet и т.д.) при регистрации. Можно рассмотреть использование *входных данных* через прокси вроде OpenRouter, где есть `Anthropic Claude free` или через облака (например, HuggingFace реплика модели).  

- **OpenAI** – бесплатный начальный кредит при регистрации (обычно $18), далее платно. GPT-3.5 Turbo при небольших запросах можно считать «дешёвым» вариантом (USD0.002/1K токенов). Для CLI-проектов обычно достаточно gpt-3.5 для локальных задач, а GPT-4 использовать экономно (например, через редкий «инструментальный» вызов).  

При выборе провайдера важно учитывать ограничения: многие требуют привязки номера телефона или верификации (NVIDIA, Mistral, Alibaba). Также проверьте, поддерживают ли эти провайдеры **вызов функций (function calling)**. Некоторые шлюзы (OpenRouter, HuggingFace) позволяют пересылать параметры вызовов как есть, но для OpenAI/Claude нужно давать данные в нужном формате (JSON-функций).  

## Конкретные планы по интеграции агентов  

Ниже приводятся шаги по добавлению каждого выбранного агента в проект. Для каждого – установка, проверка зависимостей и добавление в меню CLI:  

- **OpenAI Codex CLI** – официальный CLI-агент от OpenAI (Runs on GPT-4o/Codex). Установка простая:  
  ```bash
  # Linux/macOS: установить Codex CLI через официальный скрипт
  curl -fsSL https://chatgpt.com/codex/install.sh | sh
  ```  
  (для Windows добавить аналогичный вызов в PowerShell). После этого в любом терминале появится команда `codex`. В скрипте *bootstrap.sh* и *install.ps1* следует добавить эти строки и проверку, установлен ли уже `codex`. Затем в TUI-меню добавить пункт “Codex CLI” и запускать команду `codex` (она сама предложит войти в аккаунт ChatGPT). Код примера для Python-контроллера меню:  
  ```python
  # Example: launching Codex CLI
  import subprocess
  if choice == 'Codex CLI':
      subprocess.run(['codex'])
  ```  
  Подключаемые модели: GPT-4o, GPT-3.5-turbo. **Ограничения**: требует аккаунт ChatGPT Plus/Pro/Business.  

- **Google Gemini CLI** – терминальный агент Google на базе Gemini. Установка через npm или Homebrew:  
  ```bash
  # Установить Gemini CLI глобально
  npm install -g @google/gemini-cli
  ```  
  (на Mac/Linux можно также `brew install gemini-cli`). В проекте это команда для установки должна быть добавлена (в *bootstrap.sh* и PowerShell-скрипт). После установки команда `gemini` запускает интерфейс. В меню добавить пункт “Google Gemini CLI” и выполнять `subprocess.run(['gemini'])`. Модели: Gemini 1.5–3.5 (через Google AI Studio потребуется API-ключ или OAuth). **Ограничения**: для новых версий Gemini (3.5) нужна активная учётная запись Google Cloud AI Studio, есть лимиты бесплатных запросов.  

- **Hermes Agent** – мультиагент NousResearch. Инсталляция по официальному Quickstart:  
  ```bash
  # Linux/macOS/WSL2: командная строка для установки Hermes
  curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash
  ```  
  (для Windows PowerShell – `iex (irm https://hermes-agent.nousresearch.com/install.ps1)`). Скрипты установки CI должны запустить этот шаг. После установки появляется команда `hermes`. В меню добавить «Hermes Agent» → `subprocess.run(['hermes', 'cli'])`. Пользователь настраивает провайдер через `hermes model`. Hermes поддерживает собственный "Nous Portal" и множество API (OpenAI, Claude, OpenRouter, HuggingFace и др.). **Ограничения**: для крупных провайдеров может понадобиться авторизация; mobile-платформы (Android) тоже поддерживаются (Termux).  

- **OpenCode** – терминальный агент от Anomaly (ранее OpenDevin). Установка по скрипту:  
  ```bash
  # Установка OpenCode CLI через скрипт
  curl -fsSL https://opencode.ai/install | bash
  ```  
  (скрипт сам предложит пути установки). После этого команда `opencode` станет доступной. В TUI добавить «OpenCode CLI» → `subprocess.run(['opencode'])`. OpenCode использует токены и API (нужен OpenAI/OpenRouter-ключ и др., см. документацию), но для базового использования бесплатен. **Ограничения**: может спрашивать токен OpenAI; есть платная версия для корпоративных.  

- **Open Interpreter** – локальный агент для выполнения кода. Установка через pip:  
  ```bash
  # Установка Open Interpreter
  pip install git+https://github.com/OpenInterpreter/open-interpreter.git
  ```  
  После этого команда `interpreter` будет доступна. В меню добавить «Open Interpreter» → `subprocess.run(['interpreter'])`. По умолчанию он спрашивает API-ключ OpenAI для моделей GPT-4o; для бесплатной работы можно настроить HuggingFace или Llama-модель (есть поддержка локальных LLM). **Ограничения**: если использовать GPT-4o, нужны ключи OpenAI. Локальные модели требуют достаточного GPU/RAM.  

- **Aider** – CLI-агент для парного программирования. Установка по скрипту:  
  ```bash
  # Mac/Linux установка Aider
  curl -LsSf https://aider.chat/install.sh | sh
  ```  
  (для Windows PowerShell: `powershell -ExecutionPolicy ByPass -c "irm https://aider.chat/install.ps1 | iex"`). Команда `aider` появится в системе. Добавьте в меню «Aider CLI» → `subprocess.run(['aider'])`. Aider поддерживает DeepSeek, Anthropic, OpenAI и др. модели через параметры `--model` и ключи API. **Ограничения**: требует Python 3.8–3.13.  

- **Qwen Code CLI** – терминальный агент Alibaba. Установка через скрипт:  
  ```bash
  # Установка Qwen Code CLI (Linux/macOS)
  curl -fsSL https://qwen-code-assets.oss-cn-hangzhou.aliyuncs.com/installation/install-qwen-standalone.sh | bash
  ```  
  (для Windows PowerShell: `irm ... | iex`). После этого команда `qwen` будет доступна. В меню добавить «Qwen Code CLI» → `subprocess.run(['qwen'])`. Qwen Code из коробки оптимизирован под модели Qwen3-Coder, но может работать и с OpenAI/Gemini через маршрут OpenAI-совместимых ключей. **Ограничения**: бесплатный уровень требует ключа Alibaba Cloud или OpenRouter (см. новости в README).  

- **Дополнительные агенты (по желанию):** если хочется ещё больше покрытия, можно добавить **Pi (pi-cli)**, **Goose**, **Claurst** и др. Их установка обычно описана в документации GitHub (часто через `pip install pi-cli`, `cargo install claurst`, `npm i -g goose-agents` и т.п.). Убедитесь, что эти агенты поддерживают модели/инструменты, которые вам нужны.  

При интеграции каждого агента следует учесть:  
- **Зависимости**: поставить нужные библиотеки (Python, Node.js, Rust и т.д.). Например, Codex CLI и Gemini требуют Node.js; Hermes, Open Interpreter, Aider – Python; Claw Code (если нужен) – Rust. Добавьте эти пакеты в установщики.  
- **Библиотеки**: использовать английские комментарии в коде, но документация/пример кода на русском в ответе.  
- **Интеграцию в TUI**: расширить меню новыми пунктами. Если TUI реализован на Bash/Python/Go, вставьте новые блоки `case`/`if` для обработки выбора. Пример на псевдо-Python (см. выше) показывает подход.  
- **Прокси/прокладки**: некоторые агенты могут требовать отдельные ша 
ги. Например, Claw Code по сути свободен, но для функций GPT-4o ему может понадобиться **Codex CLI** или **OpenAI API key**. Если использовать Llama- или Qwen-агента без доступа в интернет, можно подружить их с MCP-серверами или локальным `llama.cpp`. Если модель не поддерживает функцию напрямую, можно использовать **Outlines/Jsonformer** для контроля формата (см. технологию в [17†L197-L205]).  

Каждому провайдеру/модели свойственны ограничения: контекстное окно, скорость, языковые особенности. Например, YandexGPT лучше работает с русским языком, Gemini – с англ. запросами. Этот нюанс стоит отражать в инструкциях (например, что YandexGPT стоит выбрать для русскоязычных задач).  

## Рекомендации по улучшению и чек-лист качества  
Чтобы проект был «идеальной конфеткой», нужно усилить следующие моменты:

1. **Чистая архитектура скриптов**: разбейте большой `bootstrap.sh`/`install.ps1` на логические модули – функции для каждой группы агентов. Это упростит добавление новых. Убедитесь, что повторяющиеся действия (например, проверка наличия `pip`, `npm`) оформлены в функции. Покройте скрипты тестами (bash — unit-тесты с `bats`, PowerShell — Pester). Настройте GitHub Actions для автоматической проверки (shellcheck/PSScriptAnalyzer, синтаксический анализ).

2. **Конфигурация провайдеров**: храните API-ключи и параметры в конфигурационных файлах (например, `$HOME/.cli-codes/config.json`). Это упростит добавление новых провайдеров (например, Yandex, Azure, AWS). Проверьте, что TUI позволяет вводить ключи при первом использовании или считывать их из окружения. Добавьте проверку валидности ключа перед запуском агента.

3. **Доступность TUI**: улучшите навигацию. Например, добавьте **поиск агентов по названию** или фильтрацию по провайдеру. Убедитесь, что после выполнения агента терминал возвращается в меню, а не просто завершается. Добавьте подсказки (tooltips) к пунктам меню: краткое описание агента и ссылку на документацию.

## Дорожная карта (Roadmap)  

1. **MVP (1–2 недели)**: интегрировать **Codex CLI, Gemini CLI, Hermes Agent**. Настроить базовую TUI с этими пунктами. Обеспечить установку через скрипты. Протестировать базовые сценарии (напр., `codex` должен открыть TUI Codex, `gemini` – Gemini и т.д.).  
2. **Расширение (2–4 недели)**: добавить **OpenCode, Aider, Open Interpreter, Qwen Code**. Обновить TUI и документацию. Начать CI-тесты: проверять установку и запуск `--help` для каждого нового агента.  
3. **Провайдеры (параллельно)**: протестировать работу с бесплатными API – зарегистрировать ключи OpenAI, Google, Yandex, Alibaba, настроить Hermes на Nous/DeepSeek. Обновить инструкции по получению ключей (желательно на русском).

6. **Долгосрочно (4–6 недель)**: опубликовать релиз, собрать фидбек, поддерживать обновления. Добавить новые агенты по запросу (например, если появятся GPT4All-CLI, xAI CLI).
